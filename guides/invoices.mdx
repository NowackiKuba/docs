# Invoices

Billy manages invoices generated by Stripe subscriptions automatically. You can also create manual invoices for one-off charges and retrieve or void existing invoices through the API.

## Creating a manual invoice

Use manual invoices for ad-hoc charges outside a subscription — setup fees, consulting hours, or usage overages.

```bash
curl -X POST https://api.usebilly.dev/api/billing/invoices \
  -H "Authorization: API Key <your_api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "customerId": "cus_abc123",
    "currency": "usd",
    "items": [
      {
        "description": "Onboarding setup fee",
        "amount": 50000
      }
    ]
  }'
```

<Note>
  Amounts are in the smallest currency unit (cents for USD). `50000` = $500.00.
</Note>

## Retrieving invoices

Fetch invoices for a specific customer or across an entire project.

```bash
# By customer
curl "https://api.usebilly.dev/api/billing/invoices/customer/{customerId}" \
  -H "Authorization: API Key <your_api_key>"

# By project
curl "https://api.usebilly.dev/api/billing/invoices/project/{projectId}" \
  -H "Authorization: API Key <your_api_key>"
```

Both endpoints support pagination (`limit`, `offset`, `cursor`) and filtering by `status`, `currency`, and date range (`from`, `to`).

### Invoice statuses

| Status | Description |
|--------|-------------|
| `draft` | Invoice exists but has not been finalized. Can be deleted. |
| `open` | Invoice has been sent to the customer. Can be voided. |
| `paid` | Invoice has been paid. Immutable. |
| `void` | Invoice has been voided. Immutable. |
| `uncollectible` | Payment has been abandoned. |

## Voiding an invoice

Voiding cancels an open invoice without deleting the record. The invoice status changes to `void` and no further payment attempts are made.

```bash
curl -X PATCH https://api.usebilly.dev/api/billing/invoices/{id}/void \
  -H "Authorization: API Key <your_api_key>"
```

<Warning>
  Voiding is irreversible. Once an invoice is voided, it cannot be reopened or paid. Only invoices with status `open` can be voided.
</Warning>

## Deleting an invoice

Hard deletion is only permitted for invoices in `draft` status.

```bash
curl -X DELETE https://api.usebilly.dev/api/billing/invoices/{id} \
  -H "Authorization: API Key <your_api_key>"
```

<Note>
  Paid invoices and voided invoices follow the soft-delete pattern and cannot be physically deleted. This protects the integrity of your financial records.
</Note>

## Payment methods

List a customer's saved payment methods to display them in your UI or verify that a customer has a valid payment method before creating a subscription.

```bash
curl "https://api.usebilly.dev/api/billing/payment-methods/{customerId}" \
  -H "Authorization: API Key <your_api_key>"
```

Remove a specific payment method:

```bash
curl -X DELETE https://api.usebilly.dev/api/billing/payment-methods/{id} \
  -H "Authorization: API Key <your_api_key>"
```

<Warning>
  Removing the only payment method for a customer with an active subscription may cause future renewal failures. Check that at least one valid method remains.
</Warning>
