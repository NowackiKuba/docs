# API Reference

The Billy REST API follows OpenAPI 3.0 conventions. All endpoints are authenticated with an API key and return JSON.

## Base URL

```
https://api.usebilly.dev
```

## Authentication

Include your API key in the `Authorization` header:

```http
Authorization: API Key <your_api_key>
```

## Response format

### Success

All successful responses are wrapped in a `data` envelope:

```json
{
  "data": { ... }
}
```

### Errors

Errors return a structured `error` object:

```json
{
  "error": {
    "statusCode": 404,
    "message": "Customer not found",
    "error": "Not Found",
    "code": "CUSTOMER_NOT_FOUND",
    "timestamp": "2026-04-24T10:00:00.000Z",
    "path": "/api/customers/cus_invalid"
  }
}
```

## Pagination

List endpoints support offset-based and cursor-based pagination via query parameters:

| Parameter | Type | Description |
|-----------|------|-------------|
| `limit` | integer | Number of records per page. Must be greater than 0. |
| `offset` | integer | Number of records to skip. |
| `cursor` | string | Cursor value for cursor-based pagination. |
| `orderBy` | `asc` \| `desc` | Sort direction. |
| `orderByField` | string | Field to sort by (e.g., `createdAt`). |
