# Environments

Billy enforces strict isolation between development and production data through **Environments**. Every Billy project is permanently locked into either Test Mode or Live Mode upon creation.

## Test Mode vs Live Mode

| Mode | Stripe Connection | Real Money | Sandbox Data |
|---|---|---|---|
| **Test Mode** | Stripe Test Keys | No | Yes |
| **Live Mode** | Stripe Live Keys | Yes | No |

<Note>
**Data Isolation:** Test and Live projects share absolutely zero data. Customers, subscriptions, coupons, and tokens created in a Test project cannot be migrated to a Live project.
</Note>

## Recommended Setup

For a standard SaaS application, you should maintain two Billy projects:

1. `Acme Corp (Test)`: Connected to your Stripe Test mode. Use this project's API key in your local development (`localhost`) and staging environments.
2. `Acme Corp (Production)`: Connected to your Stripe Live mode. Use this project's API key exclusively in your production environment.

## Simulated Testing

While using a Test project, Billy intercepts billing operations and routes them to Stripe's sandbox. You can use Stripe's [testing cards](https://stripe.com/docs/testing) to simulate successful payments, declines, 3D secure authentication, and fraud.

<Warning>
Webhooks configured in your Test project will only receive events generated by Test mode operations. You must configure a separate webhook endpoint for your Live project.
</Warning>
