# Billy API Reference

The Billy API exposes endpoints across seven domains. All requests require an API key and return JSON.

<Note>
  This reference is generated from the Billy OpenAPI specification. Use the endpoint pages in the sidebar to explore request parameters, response schemas, and example payloads.
</Note>

## Authentication

```http
Authorization: API Key <your_api_key>
```

API keys are scoped to a specific project. Requests made with a key belonging to Project A cannot read or write data belonging to Project B.

## HTTP methods

| Method | Usage |
|--------|-------|
| `GET` | Retrieve a resource or list of resources. |
| `POST` | Create a new resource. |
| `PATCH` | Partially update an existing resource. |
| `DELETE` | Delete a resource (soft-delete unless otherwise noted). |

## Common error codes

| Code | HTTP status | Description |
|------|-------------|-------------|
| `UNAUTHORIZED` | 401 | Missing or invalid API key. |
| `FORBIDDEN` | 403 | API key lacks permission for this operation. |
| `NOT_FOUND` | 404 | Requested resource does not exist. |
| `CONFLICT` | 409 | Unique constraint violation (e.g., duplicate Stripe ID). |
| `VALIDATION_ERROR` | 400 | Request body or query parameters failed validation. |
