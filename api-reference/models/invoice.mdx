# Invoice Object

Invoices represent financial demands generated by subscriptions, one-off charges, or manual creations.

<ResponseField name="id" type="string" required>
  The unique internal identifier for the invoice.
</ResponseField>

<ResponseField name="number" type="string">
  The sequential invoice number generated by Stripe (e.g., `INV-0001-001`). Null for draft invoices.
</ResponseField>

<ResponseField name="stripeInvoiceId" type="string">
  The corresponding Stripe `in_` ID.
</ResponseField>

<ResponseField name="customerId" type="string" required>
  The ID of the customer being billed.
</ResponseField>

<ResponseField name="projectId" type="string" required>
  The ID of the project the invoice belongs to.
</ResponseField>

<ResponseField name="status" type="string" required>
  The payment and lifecycle status of the invoice.
  <Expandable title="Allowed values">
    - `draft`: The invoice is open for modification and hasn't been finalized.
    - `open`: The invoice is finalized and awaiting payment.
    - `paid`: The invoice has been successfully paid.
    - `uncollectible`: The invoice has exhausted retries and is written off.
    - `void`: The invoice was canceled and is no longer payable.
  </Expandable>
</ResponseField>

<ResponseField name="type" type="string" required>
  The origin classification of the invoice.
  <Expandable title="Allowed values">
    - `subscription`: Generated by a recurring billing cycle.
    - `credit_note`: Represents a refund or credit adjustment.
    - `one_time`: A manually created ad-hoc invoice.
    - `trial`: A zero-dollar invoice generated at the start of a trial.
    - `past_due_recovery`: Generated during a dunning recovery flow.
  </Expandable>
</ResponseField>

<ResponseField name="currency" type="string" required>
  Three-letter ISO currency code (e.g., `usd`, `eur`).
</ResponseField>

<ResponseField name="subtotal" type="integer" required>
  Total amount before discounts and taxes (in the smallest currency unit, e.g., cents).
</ResponseField>

<ResponseField name="tax" type="integer">
  Total tax amount applied (in the smallest currency unit).
</ResponseField>

<ResponseField name="total" type="integer" required>
  Final total amount after discounts and taxes (in the smallest currency unit).
</ResponseField>

<ResponseField name="amountPaid" type="integer" required>
  The amount successfully charged for this invoice.
</ResponseField>

<ResponseField name="amountRemaining" type="integer" required>
  The remaining balance due.
</ResponseField>

<ResponseField name="hostedInvoiceUrl" type="string">
  A Stripe-hosted public URL where the customer can view and pay the invoice.
</ResponseField>

<ResponseField name="invoicePdf" type="string">
  A URL to download the PDF version of the invoice.
</ResponseField>

<ResponseField name="issueDate" type="string (ISO 8601)">
  The timestamp when the invoice was finalized (`status` changed to `open`).
</ResponseField>

<ResponseField name="dueDate" type="string (ISO 8601)">
  The deadline for payment.
</ResponseField>

<ResponseField name="periodStart" type="string (ISO 8601)">
  The start of the billing period covered by this invoice.
</ResponseField>

<ResponseField name="periodEnd" type="string (ISO 8601)">
  The end of the billing period covered by this invoice.
</ResponseField>

## Lifecycle
- **Drafts**: Can be hard-deleted via the API.
- **Open**: Can be voided.
- **Paid**: Fully immutable. Soft-deleted only.
